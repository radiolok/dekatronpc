# План работ по созданию входных файлов для DekatronPC Block Place & Route

## Обзор

Для полноценной работы инструмента Place & Route необходимы три типа входных файлов:

1. **Liberty-файл** (`.lib`) — описание ламповых логических примитивов
2. **Verilog-нетлист** (`.v`) — плоский структурный нетлист синтезированной машины
3. **KiCad-нетлисты модулей** (`.net`) — схемы печатных плат с разъёмом 2×36

Зависимость: Liberty → Verilog (синтез Yosys). KiCad-нетлисты независимы.

---

## 1. Liberty-файл (`dekatronpc.lib`)

**Что это:** описание логических ячеек (ламповых примитивов): имена и направления пинов.

**Источник:** ручное описание на основе схемотехники ламповой логики + синтез через Yosys.

### Шаги

| # | Действие | Результат |
|---|----------|-----------|
| 1.1 | Определить набор ламповых примитивов. Из wiki: 6Н16Б (сдвоенный триод, импульсный) → `TRIODE_INV`, `TRIODE_BUF`, `TRIODE_NAND2`. 6Ж2Б (пентод, две управляющие сетки) → `PENTODE_AND2`, `PENTODE_NOR2`. 6Х7Б (сдвоенный диод) → `DIODE_OR2`, `DIODE_AND2`. | Список cell-типов |
| 1.2 | Для каждого примитива определить набор пинов: сигнальные входы (сетки), выход (анод/катод), питание (накал ~6.3V, анодное +150V). | Таблица pin-ов |
| 1.3 | Определить составные макро-блоки как liberty-ячейки (из wiki: декатронная ячейка счётчика, модуль чтения 8-4-2-1, модуль записи, модуль переноса, буферы). | Расширенный `.lib` |
| 1.4 | Оформить в формате Synopsys Liberty (подмножество, достаточное для Yosys): `cell (NAME) { pin (PIN) { direction: "input"|"output"; } }` | Файл `dekatronpc.lib` |
| 1.5 | Добавить атрибут `area` для примитивов (для оптимизации площади при синтезе). | Атрибут `area` |

### Минимальный пример для тестирования

```liberty
library (DekatronPC) {
  cell (TRIODE_INV) {
    area: 1;
    pin (IN)  { direction: "input"; }
    pin (OUT) { direction: "output"; }
  }
  cell (PENTODE_AND2) {
    area: 2;
    pin (A)   { direction: "input"; }
    pin (B)   { direction: "input"; }
    pin (Y)   { direction: "output"; }
  }
  cell (DIODE_OR2) {
    area: 0.5;
    pin (A)   { direction: "input"; }
    pin (B)   { direction: "input"; }
    pin (Y)   { direction: "output"; }
  }
  cell (DECATRON_CELL) {
    area: 50;
    pin (CLK)      { direction: "input"; }
    pin (DIR)      { direction: "input"; }
    pin (SET)      { direction: "input"; }
    pin (SETZERO)  { direction: "input"; }
    pin (IN_8421_0) { direction: "input"; }
    pin (IN_8421_1) { direction: "input"; }
    pin (IN_8421_2) { direction: "input"; }
    pin (IN_8421_3) { direction: "input"; }
    pin (OUT_8421_0) { direction: "output"; }
    pin (OUT_8421_1) { direction: "output"; }
    pin (OUT_8421_2) { direction: "output"; }
    pin (OUT_8421_3) { direction: "output"; }
    pin (CARRY_IN)  { direction: "input"; }
    pin (CARRY_OUT) { direction: "output"; }
    pin (ZERO)      { direction: "output"; }
  }
}
```

## 2. Verilog-нетлист (`dekatronpc_flat.v`)

**Что это:** плоский структурный нетлист всей машины — результат синтеза behavioural Verilog в примитивы из liberty.

**Источник:** запуск Yosys с liberty-файлом на behavioural-модели (уже существует в `rtl/`).

### Шаги

| # | Действие | Результат |
|---|----------|-----------|
| 2.1 | Убедиться, что behavioural Verilog-модель (`rtl/*.v`) синтезопригодна: нет `initial`, `$display`, несинтезируемых конструкций. | Готовый к синтезу RTL |
| 2.2 | Создать скрипт для Yosys (`synth.ys`): | Скрипт синтеза |
| 2.3 | Запустить синтез: `yosys synth.ys` | Плоский нетлист `dekatronpc_flat.v` |
| 2.4 | Проверить консистентность: все имена ячеек в нетлисте должны присутствовать в `.lib` ИЛИ быть описаны как External Elements в webui. | Отчёт о несоответствиях |
| 2.5 | (Опционально) Разделить нетлист по блокам: `IpLine_flat.v`, `ApLine_flat.v`, `MachineCtrl_flat.v` — для независимой трассировки каждого из трёх 4U-блоков. | Нетлисты блоков |
| 2.6 | Создать минимальный тестовый нетлист (один декатронный счётчик, ~50 инстансов) для отладки инструмента, не дожидаясь полного синтеза. | `test_counter_flat.v` |

### Пример скрипта Yosys (`synth.ys`)

```tcl
# Read behavioural RTL
read_verilog rtl/dekatronpc_top.v
read_verilog rtl/ipline.v
read_verilog rtl/apline.v
read_verilog rtl/machine_ctrl.v
read_verilog rtl/counter.v
read_verilog rtl/memory.v

# Hierarchy
hierarchy -top dekatronpc_top

# Generic synthesis
proc; opt; fsm; opt; memory; opt

# Map to liberty cells
techmap
abc -liberty dekatronpc.lib

# Clean and export
opt_clean
write_verilog -noattr dekatronpc_flat.v
```

## 3. KiCad-нетлисты модулей (`.net`)

**Что это:** принципиальные схемы печатных плат модулей с разъёмом ГРПМШ 2×36 (A1..A36, B1..B36).

**Источник:** проектирование в KiCad (стадия 2 — прототипирование). Пока не сделано.

### Шаги

| # | Действие | Результат |

| 3.1 | Спроектировать в KiCad библиотечный символ разъёма: 2×36 контактов, A1..A36 (левая сторона), B1..B36 (правая сторона). | `.lib` / `.kicad_sym` |

## Приоритет и порядок выполнения

```
Неделя 1-2: Liberty-файл (minimal)
           │
           ├── 5-10 примитивов: TRIODE_INV, PENTODE_AND2, DIODE_OR2, DECATRON_CELL, ...
           │
Неделя 2-3: Тестовый Verilog-нетлист
           │
           ├── Один декатронный счётчик (~50 инстансов)
           ├── Отладка Yosys → Liberty маппинга
           │
Неделя 3-4: Отладка инструмента на тестовых данных
           │
           ├── Этапы 2-6 Place & Route
           │
Параллельно: KiCad-схемы модулей
           │
           ├── Логический модуль (24 мм)
           ├── Декатронный модуль (36 мм)
           ├── Модуль питания
           │
Финал:      Полный синтез всей машины
           │
           └── ~1244 лампы, ~4 счётчика, 3 блока
```

## Ключевые решения, которые нужно принять

1. **Формат liberty для ламповых примитивов:** Yosys требует функциональное описание для маппинга (`abc`). Нужно ли писать полные таблицы истинности для каждого примитива, или достаточно структурного описания?

2. **Иерархия нетлиста:** AGENTS.md говорит «плоский структурный нетлист». Но физически машина состоит из трёх блоков (IpLine, ApLine, MachineCtrl). Стоит ли сохранять иерархию для группировки модулей при размещении?

3. **Декатронная ячейка как примитив или составной блок:** В liberty это одна cell с 15+ пинами, или иерархия: декатрон + усилители + декодер? Влияет на гранулярность размещения.
